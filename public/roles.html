Who can play here

A role is not a permanent label. It is the contribution you can make right now to move a project forward.

Technical
Engineer / Developer / AI Practitioner
Build systems, automate work, structure data, ship interfaces, and create proof.
Creative
Designer / Writer / Creator
Shape clarity, user experience, storytelling, visuals, and communication.
Domain
Healthcare / Legal / Education / Business
Define the real problem, validate usefulness, and keep the work grounded.
Operations
Organizer / Coordinator / Founder
Keep momentum, align people, manage scope, and move ideas into execution.

Examples of role fit

Use these examples to see how different fields connect around one project.

Example 1
Doctor + Engineer + Designer

A healthcare workflow project needs a domain expert, a systems builder, and a clear interface.

  • Doctor defines the workflow problem
  • Engineer structures the system
  • Designer simplifies the experience
Example 2
Business + Data + Automation

A reporting or analytics project succeeds when business goals, data logic, and automation meet.

  • Business identifies the KPI
  • Data person designs the signal
  • Automation keeps it running
Example 3
Beginner + Mentor + Project Team

Beginners can still contribute by learning fast, documenting well, and shipping small proof.

  • Pick one starter repo
  • Join a sprint
  • Show one visible output
